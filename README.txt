1. 프로젝트명
=> ready-for-work

2. 개발하게 된 배경
=> 내가 아침에 출근준비를 하는데, 스마트폰으로 날씨앱을 켜서 날씨확인하고, 집 나가는 시간에 맞춰
지하철앱으로 시간표확인하고, 캘린더앱으로 일정확인하고 등 번거롭게 하나씩 확인하는게 불편하다고 느껴져서
한번에 확인할 수 있는 프로그램이 있으면 편하겠다는 생각이 듦.

3. 목적
=> 사용자가 출근시에 확인하는 기능(날씨,지하철시간표 등등)들을 한번에 확인가능한 웹앱을 개발하는 것
(웹앱을 만드는이유는 내가 java,springBoot를 주로 사용했기때문에 개발편의성때문에),
(중요!)그렇기때문에 스마트폰 브라우저에서 '홈화면에 추가'했을때 완벽하게 작동하는 모바일 최적화 웹앱을 개발하는것이 1차목표!
하지만 결국 사용성,접근성의 관점에서 앱스토어에 출시할 수 있는 앱으로 전환하는것이 최종목표.

4. 특징
- 화면은 카드형UI 로 개발: (ex. 최상단(긴급) / 중단(준비) / 하단(일정/메모) )
- 제공할 체크사항으로는 날씨,교통시간표,주요뉴스,관심자산현황(미증시,국내증시),캘린더(일정),옵시디언 메모앱
- 제공되는 체크사항들은 자유롭게 추가or제거 가능하도록 함
- 날씨는 날씨API를 기반으로 하고, 단순히 날씨만 알려주는것이 아니라 '오늘의 옷차림가이드'를 제공하는 것으로 확장성 있게 개발
- 교통시간표도 kakao또는 naver제공 API를 기반으로하고, 시간표 외에 교통이슈등이 있으면 요약해서 정보제공하도록 개발
- (중요!)이런식으로 체크하는 사항들은 급한 아침시간에 사용하는 불필요한 판단시간등을 줄여주는데 목적을 두도록 기능개발!

5. 기술스택
- BackEnd : java , springBoot
- FrontEnd : React
- 그 외 필요한 기술스택이나 버전은 클로드가 최적이라고 판단되는 것으로 결정
